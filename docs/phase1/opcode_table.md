# Opcode Table (Phase 1)

Address alignment note for 4KB memory: absolute control-flow targets are encoded in the same instruction word as `[11:0]=target_addr`.

Early decode policy for branch prediction:
- `instr[16]=1` for unconditional immediate branch only (`JMP`,`CALL`)
- `instr[17]=1` for conditional branch only (`JZ`, `JN`, `JC`)
- `RET` do not use early-decode branch bits

## User-visible ISA opcodes (assembler-supported)

| Mnemonic | Opcode (bin) | Format | Notes |
| -------- | ------------ | ------ | ----- |
| NOP      | 00000        | F0     | no operation |
| HLT      | 00001        | F0     | halt pipeline/fetch |
| SETC     | 00010        | F0     | set carry flag |
| NOT      | 00011        | F1     | one operand (mirrored register fields) |
| INC      | 00100        | F1     | one operand (mirrored register fields) |
| OUT      | 00101        | F1     | one operand (mirrored register fields) |
| IN       | 00110        | F1     | one operand (mirrored register fields) |
| MOV      | 01000        | F2-2R  | `MOV rdst, rsrc1` |
| SWAP     | 01001        | F2-2R  | cycle 1 behaves as `MOV rdst, rsrc1`; control unit injects `SWAP2` |
| ADD      | 01010        | F2-3R  | updates Z/N/C |
| SUB      | 01011        | F2-3R  | updates Z/N/C |
| AND      | 01100        | F2-3R  | updates Z/N |
| IADD     | 01101        | F3     | `rdst, rsrc1, imm16` |
| PUSH     | 10000        | F1     | stack push (`SP--`) |
| POP      | 10001        | F1     | stack pop (`SP++`) |
| LDM      | 10010        | F3     | `rdst, imm16` |
| LDD      | 10011        | F3     | `rdst, offset16(rsrc1)` |
| STD      | 10100        | F3     | `rsrc1, offset16(rsrc2)` |
| JZ       | 11000        | F3-Addr12 | conditional branch (`instr[17]=1`) |
| JN       | 11001        | F3-Addr12 | conditional branch (`instr[17]=1`) |
| JC       | 11010        | F3-Addr12 | conditional branch (`instr[17]=1`) |
| JMP      | 11011        | F3-Addr12 | unconditional branch (`instr[16]=1`) |
| CALL     | 11100        | F3-Addr12 | unconditional branch (`instr[16]=1`) |
| RET      | 11101        | F0     | return via stacked PC |
| INT      | 11110        | F3-IntIdx | software interrupt entry; pushes PC then chains to internal ops (`INT 0 -> idx 10`, `INT 1 -> idx 11`, HW interrupt marker `01`) |
| RTI      | 11111        | F0     | return from interrupt |

## Internal-only micro-operations (not assembler-supported)

These operations are generated by decode/control logic through `MULTICYCLE_SEL`. They are not valid source mnemonics in `.asm` files.

| Internal op | Internal Opcode (bin) | Triggered by | Behavior |
| ----------- | --------------------- | ------------ | -------- |
| SWAP2       | 00111                 | `SWAP`       | reverse move (`rsrc1 <- rdst_saved`) and assert `SWAP_2ND_CYCLE=1` to disable EX1->EX1 forwarding in that cycle |
| INT2        | 01110                 | `INT`        | push flags to stack |
| INT3        | 01111                 | `INT2`       | load interrupt vector address into PC |

## Encoding conventions frozen

- F0 encodes `Rdst` and `Rsrc1` as `000`.
- F1 mirrors one operand into both `Rdst[26:24]` and `Rsrc1[23:21]`.
- F2 `SWAP` uses the same register layout as `MOV` in instruction memory; second cycle work is internal (`SWAP2`).
- F3 immediate data (`IADD`, `LDM`, `LDD`, `STD`) stores signed/unsigned value in `[15:0]`.
- F3 address immediates are always in `[11:0]` within `0x000..0xFFF`.
